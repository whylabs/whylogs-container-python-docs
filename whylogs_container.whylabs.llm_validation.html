class whylogs_container.whylabs.llm_validation.metadata.ProfilingRule(metrics: List[str])#

Bases: object

A ProfilingRule for a dataset-id that indicates which of langkit’s metrics should be used for profiling and monitoring in WhyLabs. In case the user doesn’t set it, the container will only work in validation mode.

metrics is a list of langkit’s metrics that will be used for profiling and monitoring in WhyLabs. To include all of the available metrics with langkit.llm_metrics, use the keyword all. This dataclass is used from the YAML parsing phase in the container setup.

An example YAML file for this dataclass is defined below.

Example 1: ```yaml whylabs_dataset_id: model-15 policy: my_new_policy id: 9294f3fa-4f4b-4363-9397-87d3499fce28 policy_version: 1 schema_version: 0.0.1

profiling:
metrics:

  • all

```
